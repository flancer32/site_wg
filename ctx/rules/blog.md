# Правила файловой структуры блога

Path: `ctx/rules/blog.md`

## Расположение записей

Записи хранятся в каталоге `tmpl/web/<locale>/blog/YYYY/MM/`, где `<locale>` — локаль страницы (`ru`, `en`, `es` и т.п.), `YYYY` — год четырьмя цифрами, `MM` — месяц двумя цифрами. Путь к файлу определяет каноническое место записи в структуре сайта и служит метаинформацией о дате публикации.

## Именование файлов

Имена соответствуют шаблону `DD-NN-slug.html`, где `DD` — двухзначный день месяца, `NN` — порядковый номер записи в пределах дня, а `slug` — краткий идентификатор. Формат имени обеспечивает уникальность и однозначный хронологический порядок при сортировке по имени.

## Назначение блоков шаблона

Каждый файл содержит минимум два блока:

- `content` — основной HTML контент индивидуальной страницы.
- `blog_item` — HTML-аннотация, которая не рендерится на странице записи и предназначена исключительно для списка. Блок `blog_item` — единственный канал передачи HTML-аннотаций от шаблона к обработчику.

Пустой или отсутствующий `blog_item` делает запись недопустимой для обработки списка.

## Порядок записей

Хронология определяется только файловой структурой: сначала года по убыванию, затем месяца, затем файлов по убыванию имени. Никакие данные внутри файлов не влияют на порядок, только путь и имя.

## Инварианты

1. `blog_item` — единственная допустимая точка экспорта HTML-аннотаций из шаблона.
2. Файловая и именованная структура обеспечивает однозначность порядка и идентификации.
