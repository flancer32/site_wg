# Правила организации файлов и рендеринга блога

## 1. Размещение файлов

Записи блога размещаются в каталоге:

```txt
tmpl/web/lang/blog/YYYY/MM/
```

где `lang` - язык страницы (`en`, `es`, `ru`), `YYYY` - год, `MM` - месяц.
Структура каталогов определяет хронологический порядок материалов.

---

## 2. Формат имени файла

Каждая запись хранится в виде Nunjucks-шаблона с именем:

```text
DD-NN-slug.html
```

где:

- `DD` - день месяца (две цифры),
- `NN` - порядковый номер события в пределах дня,
- `slug` - краткое название записи.

Формат имени должен обеспечивать однозначную идентификацию записи и соответствовать хронологии.

---

## 3. Структура шаблона записи

Каждый файл содержит:

1. основной контент записи в блоке `content`;
2. отдельный блок аннотации:

```text
{% block blog_item %}
<li class="blog-item">…</li>
{% endblock %}
```

Блок `blog_item` является частью записи, не отображается на индивидуальной странице и используется только при формировании списка событий.

---

## 4. Требования к индивидуальной странице

При рендеринге страницы должна быть доступна только основная часть записи.
Аннотация (`blog_item`) не должна присутствовать в выводе страницы и не участвует в визуальном представлении поста.

---

## 5. Требования к итоговой странице списка блога

Итоговая страница блога должна удовлетворять следующим условиям:

- содержать аннотации всех записей, размещённых в структуре `tmpl/web/ru/blog/YYYY/MM/`;
- располагать аннотации в хронологическом порядке, определяемом каталогами и именами файлов;
- извлекать контент аннотаций из блока `blog_item` каждого шаблона записи;
- формировать итоговый список в рамках основного шаблона страницы блога.

Итоговая страница не должна зависеть от внутренней разметки отдельных записей, кроме стандартизированного блока `blog_item`.

---

## 6. Общие требования

- Каждая запись является самостоятельным шаблоном.
- Формат имени и структура каталогов обеспечивают стабильность ссылок.
- Блок `blog_item` является обязательной частью каждой записи.
- Итоговый список строится исключительно на основании данных, присутствующих в шаблонах записей.
