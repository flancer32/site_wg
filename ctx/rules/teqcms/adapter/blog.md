# Обработчик данных для `/blog`

Path: `ctx/rules/teqcms/adapter/blog.md`

## Область ответственности

Обработчик предоставляет данные для индексной страницы `/blog`. Он не занимается маршрутизацией, не формирует HTML страниц и не изменяет стандартный рендер — его единственная задача собрать аннотации из шаблонов и вернуть сериализуемую структуру.

## Источники и обход

- Корень файлового дерева соответствует `tmpl/web/<locale>/blog/`, подробное описание структуры и наименований содержится в `ctx/rules/blog.md`.
- Обход выполняется по каталогам `YYYY/MM`, сортировка происходит по убыванию: сначала года, затем месяцы, затем имена файлов. Файлы вне указанного формата игнорируются.
- Каждая встреченная запись допускается только при наличии блока `{% block blog_item %}`; содержимое блока переносится в результат без нормализации или изменений.

## Формат результата

На каждую валидную запись создаётся объект:

```js
{
  slug: "<имя-файла-без-.html>",
  url: "/<locale>/blog/YYYY/MM/<file>.html",
  html: "<сырой HTML блока blog_item>"
}
```

Собранные объекты возвращаются в массиве, порядок которого соответствует порядку обхода (`YYYY` → `MM` → `file`).

## Инварианты

1. Обработчик — единственный источник данных для `/blog`.
2. `blog_item` — единственный канал передачи HTML-аннотации из файлов в обработчик.
3. Содержимое `blog_item` не модифицируется обработчиком.
4. Порядок записей определяется лишь файловой структурой и именами файлов.
