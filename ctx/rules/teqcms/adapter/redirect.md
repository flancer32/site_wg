# Обработчик переноса маршрутов (redirect-map)

**Path:** `ctx/rules/teqcms/adapter/redirect.md`

## Область ответственности

Обработчик подменяет путь запроса согласно карте переноса, обеспечивая доступ к страницам, чей адрес изменён.
Он не выполняет HTTP-редиректы, не влияет на выбор шаблонов и не обрабатывает статические ресурсы.

---

## Источник данных

Файл:

```
etc/redirect-map.json
```

Формат:

```json
{
  "/foo.html": "/blog/2025/09/05-01-abdsm-habr.html",
  "/blog/2024/legacy-post.html": "/blog/2025/updated-post.html"
}
```

Ключ — старый путь без локали,
значение — новый путь без локали.
Путь к файлу вычисляется относительно корня приложения, определённого конфигурацией.

---

## Правила обработки

1. Обработчик применяется только к HTML-маршрутам.
2. Сопоставление выполняется по `cleanPath`, полученному от CMS-хэлпера.
3. При нахождении соответствия в запрос подставляется новый путь, при этом сохраняются локаль и query-параметры.
4. Отсутствие записи в карте означает отсутствие изменений в запросе.

---

## Формат результата

Обработчик не возвращает данных: он модифицирует `req.url` и передаёт управление адаптеру.

---

## Инварианты

1. `redirect-map.json` — единственный источник правил переноса.
2. Подстановка выполняется только на уровне путей, без генерации redirect-ответов.
3. Статика не обрабатывается.
4. Локаль всегда сохраняется.
