# Итерация

- Цель: оценить текущую коммерческую упаковку `contact.html`, интерес холодного клиента к новому 3-шаговому предложению по `GitHub Flows` и предложить мягкие стартовые цены.

## Выполнено

- Просмотрена текущая русская страница контактов: `tmpl/web/ru/contact.html`.
- Сверен продуктовый контекст в:
  - `ctx/docs/product/overview.md`
  - `ctx/docs/product/pages/home.md`
- Собраны внешние ориентиры по спросу и рискам вокруг GitHub workflows и agentic workflows.
- Сформирована коммерческая оценка нового оффера для холодного клиента.
- Сформированы рекомендуемые стартовые диапазоны цен без агрессивного завышения.

## Выводы

- Текущая `contact.html` продаёт абстрактный консалтинг и ADSM, а не текущий главный продукт `GitHub Flows`.
- Для холодного клиента более жизнеспособен 3-шаговый оффер:
  - тестовое подключение репозитория к инфраструктуре Алекса;
  - развёртывание аналогичной инфраструктуры на VPS клиента;
  - помощь в настройке собственных workflow.
- На холодном рынке это может быть интересно узкому, но реальному сегменту:
  - solo founders;
  - small engineering teams;
  - technical product owners, already living in GitHub.
- Сильная сторона предложения: низкий порог входа и наблюдаемый результат внутри GitHub.
- Слабая сторона предложения: рынок ещё не знает категорию, поэтому продавать нужно не “инфраструктуру” и не “агентность”, а безопасный первый практический результат.

## Внешние опоры

- Исследование о GitHub Actions показывает, что около 30% популярных репозиториев уже используют GitHub Actions, а разработчики часто просят помощь с внедрением: `https://arxiv.org/abs/2206.14118`
- Исследование восприятия GitHub Actions показывает, что 60.87% опрошенных считают YAML-конфигурации сложными и подверженными ошибкам: `https://arxiv.org/abs/2303.04084`
- Исследование по Agentic Workflow Injection показывает новый класс рисков в AI-агентных GitHub workflows, что усиливает ценность аккуратного пилотного внедрения: `https://arxiv.org/abs/2605.07135`
- Бенчмарк реальных freelance software tasks показывает широкий рыночный разброс оплаты задач, от мелких недорогих работ до крупных внедрений: `https://arxiv.org/abs/2502.12115`
