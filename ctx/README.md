## Purpose

This directory contains the project cognitive context.

The cognitive context is the authoritative project knowledge space used by humans and agents when maintaining the project.

## Entry Points

- Read `AGENTS.md` for cognitive-context structure and governance rules.
- Read `docs/filesystem.md` for the top-level repository map.
- Read `docs/` for declarative project documentation.
- Read `agent/` only when working with repository-local agent operations or workflow assets.

## Boundary

This directory defines project knowledge and agent-operational context.

It does not replace product source code, runtime configuration, deployment secrets, or generated build artifacts.
