# Адаптер TeqCMS

Path: `ctx/rules/teqcms/adapter.md`

## Назначение

Адаптер (`App_Back_Di_Replace_Adapter`) переопределяет стандартный `Fl32_Cms_Back_Di_Replace_Adapter`, чтобы дополнить рендер только в тех маршрутах, которые требуют расширенных данных. Основной поток обработки и структура данных остаются неизменными за пределами целевых маршрутов.

## Подключение

Конфигурация заменяет стандартный адаптер:

```js
replace.add("Fl32_Cms_Back_Api_Adapter", "App_Back_Di_Replace_Adapter");
```

## Целевые маршруты

Расширение реализуется исключительно для маршрутов, имеющих отдельный документ `ctx/rules/teqcms/adapter/<route>.md`. Расширение не описывает внутреннюю логику конкретного маршрута и не обрабатывает шаблоны напрямую. Для `/blog` используется `ctx/rules/teqcms/adapter/blog.md`.

## Расширение данных

- `getRenderData` делегирует базовое поведение стандартному адаптеру и не модифицирует существующие поля.
- Дополнительные данные добавляются только в контексте целевого маршрута после вызова специализированного обработчика.
- Любые внешние источники данных описываются в документах маршрутов, а не в самом адаптере.

## Инварианты

1. Адаптер — единственная точка расширения рендера.
2. Вне маршрутов, описанных в `ctx/rules/teqcms/adapter/`, CMS работает стандартным образом.
3. Адаптер не интерпретирует шаблоны и не читает содержимое файлов.
4. Все дополнительные поля формируются обработчиками, документированными вместе с маршрутом.
