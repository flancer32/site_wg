# Конфигурирование TeqCMS в проекте

Path: `ctx/rules/teqcms/config.md`

## Правило

В проекте используется **конфигурирование через файл `teqcms.config.mjs` в корне проекта**.
TeqCMS должен обнаружить этот файл автоматически и вызвать его экспорт `configure()`.

Конфигурация обязана обеспечить два результата:

1. **Проект подключён к DI через namespace-корень**

   - префикс: `App_`;
   - путь: абсолютный каталог `src` проекта.

2. **Заменён стандартный адаптер CMS**

   - было: `Fl32_Cms_Back_Api_Adapter`;
   - должно стать: `App_Back_Di_Replace_Adapter`.

Это единственный способ конфигурирования, используемый в проекте.
