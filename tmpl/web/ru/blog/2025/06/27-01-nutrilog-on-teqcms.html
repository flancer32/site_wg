NutriLog переведён с Astro на TeqCMS

Дата публикации: 2025-06-27

С 27 июня веб-сайт NutriLog больше не использует Astro. Я полностью перевёл его на TeqCMS — файловую CMS, реализованную в соответствии с философией TeqFW.

Цель перехода — упростить управление сайтом и интегрировать автоматические переводы через API языковых моделей, таких как DeepSeek. Теперь я могу создавать и редактировать страницы локально, на своём ноутбуке, запускать перевод контента и публиковать результат через GitHub. Контент в Git-репозитории автоматически передаётся на сервер с помощью GitHub Actions.

Вместо статической генерации (SSG) теперь используется серверная сборка страниц при запросе. Это не подходит для сайтов с высокой нагрузкой, но даёт мне, как разработчику, максимальную гибкость и простоту. Клиентской части у сайта нет — всё работает на серверной стороне, с генерацией через Mustache-шаблоны.

TeqCMS в этом проекте стал не просто движком генерации, а частью архитектурного эксперимента: как может выглядеть управление контентом в условиях полной автоматизации локализации, SSR и контроля версий.
