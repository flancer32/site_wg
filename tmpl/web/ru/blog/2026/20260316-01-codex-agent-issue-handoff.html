Codex-агент и GitHub issue

16 марта 2026

16 марта Codex-агент, работавший над runtime-конфигурацией для npm-пакета @flancer32/teq-web, сначала сверил задачу с примером из спецификации, а затем показал, что прямое копирование схемы ломает тесты из-за ограничений в другом пакете @teqfw/di, где уже были добавлены лишние проверки.

После этого задача была оформлена как GitHub issue: teqfw/di#33 . Такой формат сохранил контекст и убрал лишний ручной пересказ между агентами.

По затратам это стоило только одного уточнения и одной нормальной постановки задачи. В остальном работа свелась к тому, что агент сам обнаружил ограничение, сам сформулировал причину и сам вынес исправление в отдельную задачу.

Вывод получился простой: GitHub issue удобно работает как интерфейс между агентами, если они умеют не только писать код, но и формулировать техническую задачу. Дальше остаётся автоматизировать приём таких задач серверным Codex-агентом и замкнуть цикл до конца.
