Управляемая агентная разработка веб-приложений с ИИ

Я выстраиваю процессы на базе GitHub, в которых изменение продукта проходит путь от задачи через выполнение агентом, запрос на изменение, проверку и публикацию под документированным контролем.

Посмотреть публичное демо Обсудить пилотный проект

Публичное доказательство: задача в GitHub → ИИ-агент → опубликованная страница

На TeqFW Demo Pages любой пользователь GitHub может создать задачу с запросом на демонстрационную страницу. Если запрос укладывается в документированные границы, GitHub Flows запускает агента, агент создаёт ограниченный артефакт, и результат публикуется на сайте.

Это не просто элемент портфолио. Это рабочее публичное доказательство того, что развитие продукта может идти через задачи GitHub, документированные ограничения, агентное исполнение, проверку и публикацию.

Стек метода

ADSM — это метод.

Agent Driven Software Management определяет, как контекст, документация, ограничения, проверка и доказательная база удерживают агентную разработку под контролем.

GitHub Flows — это слой исполнения.

Он связывает события GitHub, задачи, метки, запросы на изменение, серверное выполнение агентов, комментарии и проверку в единый управляемый процесс.

Tequila Framework — это прикладная платформа.

Она даёт основу на JavaScript для веб-приложений, удобных для агентной разработки: DI, модульную структуру, веб-сервер, конфигурацию, логирование, слой хранения данных и сопутствующие компоненты.
