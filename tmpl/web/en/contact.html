What is included in a pilot

A pilot is intentionally limited. It is not unlimited consulting and not a promise to automate the whole product.

  • one repository;
  • one workflow;
  • one task type;
  • one or two agent execution attempts;
  • one reviewable result;
  • short final notes with findings and next steps.

If the repository is not ready for this workflow, the result may be a clear explanation of what must be changed first.

What to include in your message

  • repository type: public, private, new, or existing;
  • what kind of task you want agents to process;
  • whether you already use GitHub Issues and Pull Requests;
  • whether you want a review, pilot, or repository setup;
  • whether the result can be described publicly as an anonymized case study.
